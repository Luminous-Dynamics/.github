## üõ†Ô∏è Technical Setup

### Prerequisites

Different projects have different requirements:

**Python Projects** (luminous-nix, kosmic-lab, webpilot):
```bash
# Python 3.11+
python --version

# Poetry (recommended) or pip
poetry install
# OR
pip install -r requirements.txt
```

**Rust Projects** (Mycelix-Core, nsfw):
```bash
# Rust toolchain
rustup --version

# Build
cargo build --release
cargo test
```

**TypeScript/JavaScript** (terra-atlas, Mycelix-Music):
```bash
# Node.js 18+
node --version

# Install and run
npm install
npm run dev
```

**NixOS Projects** (luminous-nix, nixite):
```bash
# Nix package manager
nix --version

# Enter development shell
nix develop
```

### Development Environment

1. **Clone the repository**:
   ```bash
   git clone https://github.com/Luminous-Dynamics/[project-name].git
   cd [project-name]
   ```

2. **Install dependencies** (see project README)

3. **Run tests**:
   ```bash
   # Python
   pytest

   # Rust
   cargo test

   # Node/TypeScript
   npm test
   ```

4. **Start development**:
   Check each project's README for specific instructions

## ‚úÖ Your First Contribution

Ready to make your first contribution? Here's a proven process:

### 1. Find an Issue
- Look for `good-first-issue` labels
- Check `help-wanted` labels
- Ask in discussions: "What's a good first contribution?"

### 2. Understand the Context
- Read the issue description carefully
- Review related code if it's a bug fix
- Ask clarifying questions if anything is unclear
- Check if anyone else is already working on it

### 3. Fork and Branch
```bash
# Fork the repository on GitHub, then:
git clone https://github.com/YOUR_USERNAME/project-name.git
cd project-name
git checkout -b feature/your-feature-name
```

### 4. Make Your Changes
- Write clean, intentional code
- Follow the project's coding style
- Add tests if applicable
- Update documentation

### 5. Test Thoroughly
- Run all existing tests
- Add new tests for your changes
- Test edge cases
- Verify nothing breaks

### 6. Submit Pull Request
- Push your branch to your fork
- Open a PR with clear description
- Link to the related issue
- Fill out the PR template completely
- Be responsive to review feedback

### 7. Celebrate! üéâ
- Your first contribution is complete!
- Thank the reviewers
- Help others with their first contributions

## ‚ùì Common Questions

**Q: Do I need to understand ERC philosophy to contribute?**
A: Not required, but encouraged! Understanding our values helps align contributions with our mission.

**Q: What if I break something?**
A: Don't worry! That's what tests and code review are for. We're here to help you learn.

**Q: Can I contribute without coding?**
A: Absolutely! Documentation, testing, community support, translations, and design are all valuable.

**Q: How long does PR review take?**
A: Usually 2-5 days. Be patient - we're volunteers operating on gift economy principles.

**Q: What if my idea doesn't align with ERC?**
A: Open a discussion first! We can explore how to align it or whether it fits our direction.

**Q: Do you pay contributors?**
A: We operate on gift economy principles, not traditional employment. See our [gift economy model](./CONTRIBUTING.md#-gift-economy-model).

**Q: Is this suitable for beginners?**
A: Yes! We welcome all skill levels and provide mentorship for newcomers.
