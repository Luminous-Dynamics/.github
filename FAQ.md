## Technical Questions

### Why Byzantine-resistant federated learning?

Federated learning enables ML without sharing raw data (privacy!), but it's vulnerable to poisoning attacks. Our Mycelix Protocol achieves 100% Byzantine detection, making it safe for production use in healthcare, finance, and sensitive applications.

### What is Holochain and why do you use it?

Holochain is agent-centric distributed computing that puts users in control of their data. Unlike blockchain, it's lightweight, scalable, and aligns with our consciousness-first principles of user sovereignty.

### Why NixOS?

NixOS's declarative, reproducible approach to system configuration aligns with our values of transparency and reliability. Luminous Nix makes this power accessible through natural language.

### What is MATL/0TML?

Mycelix Adaptive Trust Layer (MATL) and Zero-Trust ML (0TML) - our reputation-based system for evaluating participant trustworthiness in federated learning without surveilling data content.

### How do you ensure security?

- Regular security audits
- Dependency scanning
- Code review for sensitive changes
- Responsible disclosure policy
- End-to-end encryption where appropriate
- See [SECURITY.md](./SECURITY.md) for details

### What's the tech stack for [specific project]?

Check individual repository README files for detailed tech stack information. Each project documents its dependencies, requirements, and architecture.

### Can I self-host these applications?

Most projects are designed for self-hosting! Check individual repos for deployment guides. We encourage user sovereignty and data ownership.
