## Technical Terms - General

### Agent-Centric Architecture
Design paradigm where each user has their own "agent" (data and processing power) rather than relying on centralized servers. Core to Holochain and Mycelix systems.

### Decentralized / Distributed Systems
Systems with no single point of control or failure. Power and data distributed across participants. Examples: Mycelix ecosystem, Holochain apps.

### End-to-End Encryption (E2EE)
Encryption where only communicating users can read messages. No intermediary (not even the platform) can decrypt. Used in Mycelix Mail.

### Federated Learning
Machine learning approach where model training happens across multiple decentralized devices/servers, each holding local data samples, without exchanging the data itself.

### Holochain
Agent-centric distributed computing framework that puts users in control of their data. Used for Mycelix Mail, Music, Marketplace. Alternative to blockchain that's more scalable and energy-efficient.

### Monorepo
Single repository containing multiple related projects. Our luminous-dynamics repo contains The Weave, Sacred Core API, Field Visualizer, Codex, and more.

### P2P (Peer-to-Peer)
Network where participants interact directly without central intermediaries. Examples: Mycelix Music, Mycelix Marketplace.

### Trust Layer
System for evaluating participant trustworthiness. Our MATL (Mycelix Adaptive Trust Layer) assesses reputation without surveilling content.

### WSL2 (Windows Subsystem for Linux 2)
Allows running Linux directly on Windows. NSFW uses this to bridge Windows to Nix ecosystem.

## Mycelix Ecosystem

### Byzantine Detection
Identifying malicious participants in federated learning. Mycelix-Core achieves 100% detection at 0.7ms latency using the Krum algorithm.

### Krum Algorithm
Aggregation method for federated learning that selects gradients closest to peer consensus, rejecting outliers. Enables our Byzantine resistance.

### MATL (Mycelix Adaptive Trust Layer)
Our reputation-based system for evaluating sender/participant trustworthiness without surveilling data content. Used in Mycelix Mail and Protocol.

### Mycelix Protocol / Mycelix-Core
Production-grade Byzantine-resistant federated learning system. Implements MATL/0TML trust layer with RB-BFT consensus on Holochain.

### RB-BFT (Reputation-Based Byzantine Fault Tolerance)
Consensus mechanism combining Byzantine fault tolerance with reputation scoring. Achieves faster consensus by weighing trusted participants more heavily.

### 0TML (Zero-Trust Machine Learning)
Trust verification approach that validates participant behavior without access to training data content. Acronym: zero-Trust ML.

## NixOS & Development Tools

### Declarative Configuration
System configuration specified by declaring desired state rather than scripting steps to achieve it. Core NixOS principle.

### Nix
Functional package manager enabling reproducible, declarative system configuration. Foundation of NixOS.

### NixOS
Linux distribution built on Nix package manager. Uniquely reproducible and declarative.

### Nixite
Visual package discovery tool for NixOS. Makes exploring 70,000+ Nix packages accessible through intuitive UI.

### NSFW (Nix Subsystem for Windows)
Bridge bringing Nix's 70,000+ packages to Windows via WSL2. 500-1000Ã— faster package search than standard Nix queries.

### Reproducibility
Ability to exactly recreate system state or build. Nix/NixOS guarantee bit-for-bit identical rebuilds.
